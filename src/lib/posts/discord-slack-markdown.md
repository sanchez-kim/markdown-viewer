## 메신저 속 마크다운

디스코드(Discord)와 슬랙(Slack)은 메시지에 마크다운 스타일의 서식을 지원합니다. 완전한 마크다운은 아니지만, 핵심 서식을 알아두면 협업 메시지를 훨씬 읽기 좋게 만들 수 있습니다.

## 디스코드 서식

디스코드는 마크다운과 거의 같은 문법을 씁니다.

-   `**굵게**`, `*기울임*`, `__밑줄__`, `~~취소선~~`
-   인라인 코드는 백틱 하나(`` `코드` ``), 코드 블록은 백틱 세 개
-   인용은 `>` , 스포일러는 `||가림||`

코드 블록에 언어를 지정하면 디스코드에서도 문법 강조가 적용됩니다.

````
```python
print("Hello, Discord!")
```
````

디스코드는 2023년부터 **제목**도 지원합니다. `#`, `##`, `###` 세 단계이며 기호 뒤에 공백이 필요합니다. 글머리 목록(`-` 또는 `*`), 번호 목록(`1.`), 작게 표시되는 서브텍스트(`-#`)도 쓸 수 있습니다.

한 가지 주의할 점은 **마스크 링크**(`[보일 텍스트](주소)`)입니다. 디스코드는 피싱 방지를 위해 이 문법을 **일반 사용자 메시지에서는 막아두었고**, 봇 메시지나 임베드에서만 동작합니다. 일반 채팅에 링크를 넣으면 주소가 그대로 노출됩니다.

## 슬랙 서식

슬랙은 디스코드와 조금 다릅니다. 별표 하나로 굵게, 밑줄 기호로 기울임을 표현합니다.

-   `*굵게*` (별표 하나)
-   `_기울임_` (밑줄 하나)
-   `~취소선~` (물결 하나)
-   인라인 코드 `` `코드` ``, 코드 블록은 백틱 세 개
-   인용은 `>`

슬랙은 표준 마크다운의 굵게(`**`)가 아니라 별표 하나(`*`)를 쓰는 점만 주의하면 됩니다. 별표 두 개(`**`)는 굵게가 되지 않고 글자 그대로 보입니다.

가장 중요한 차이가 하나 더 있습니다. **슬랙의 코드 블록은 문법 강조(색상)를 지원하지 않습니다.** 백틱 세 개로 감싸도 단색 고정폭 글자로만 표시됩니다. 색이 입혀진 코드를 넣고 싶다면 일반 메시지 대신 **스니펫** 기능으로 언어를 지정해야 합니다. 코드 블록에 언어별 색이 들어가는 디스코드와는 정반대입니다.

또한 슬랙은 표준 마크다운이 아니라 **mrkdwn**이라는 자체 형식을 씁니다. 그래서 `[텍스트](주소)` 형태의 링크 문법은 통하지 않습니다. 주소를 붙여넣으면 자동으로 링크가 되고, 텍스트에 링크를 거는 별도 기능(서식 도구의 링크 버튼)으로 연결합니다.

## 실전 활용

-   **버그 공유**: 에러 메시지를 코드 블록으로 감싸면 가독성이 크게 올라갑니다.
-   **명령어 안내**: 설치/실행 명령을 인라인 코드로 표시해 오타를 줄입니다.
-   **강조**: 마감이나 중요 공지는 굵게로 처리해 놓치지 않게 합니다.

## 한눈에 보는 차이

| 서식 | 디스코드 | 슬랙 |
| --- | --- | --- |
| 굵게 | `**굵게**` | `*굵게*` |
| 기울임 | `*기울임*` | `_기울임_` |
| 취소선 | `~~취소선~~` | `~취소선~` |
| 코드 블록 강조 | 지원 | 미지원(스니펫 사용) |
| 제목(`#`) | 지원(3단계) | 미지원 |

## 정리

메신저 서식은 사소해 보이지만, 코드 한 줄을 코드 블록으로 감싸는 것만으로도 팀의 소통 효율이 올라갑니다. 플랫폼마다 미묘한 차이가 있으니 굵게 표기법만 기억해두면 헷갈리지 않습니다.
